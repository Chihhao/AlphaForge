# 名詞解釋 (Glossary) 卡片內容規範

為了確保 AlphaForge 平台內所有的「說明卡片 (EducationalHint / Glossary)」在視覺與閱讀體驗上保持高度一致，新增或修改 `glossary.json` 時，請務必遵守以下規範：

## 1. 標題與名詞 (Term) 格式
**強制規定**：所有專業名詞、指標名稱的 `term` 以及卡片內容的 Markdown 標題（即 `content` 欄位中 `##` 開頭的標題），必須統一採用 **「中文名稱 (英文名稱)」** 的格式。

✅ 正確範例：
- `RSI 指標 (Relative Strength Index)`
- `移動平均線 (Moving Average)`
- `KD 指標 (Stochastic Oscillator)`
- `乖離率 (Bias)`

❌ 錯誤範例（切勿使用）：
- `RSI 指標 (相對強弱指標)` -> 請將中文放在括號外，英文全名放在括號內
- `MA (移動平均線)` -> 請交換中英文位置
- `KD 指標` -> 缺少括號內的英文全名

## 2. 應用範圍
這項規則適用於 `/backend/app/data/glossary.json` 檔案中每一筆物件的：
1. **`term` 屬性**：這會直接影響搜尋或列表的顯示名稱。
2. **`content` 屬性中的卡片大標題**：如 `## RSI 指標 (Relative Strength Index)`，這會直接渲染在畫面的提示卡片最上方。

## 3. UI 顯示規範 (針對前端)
當在前端 UI (例如：股票詳細資訊頁面的指標卡片、獨立數據欄位) 在文字旁配置問號 `(?)` 圖示 `(<EducationalHint glossaryId="..." />)` 時：

- UI 標籤的名稱應清晰呈現該數據代表的意義。
- 在有時間週期的技術指標名稱前面，請盡可能以阿拉伯數字**附上具體天數週期**，以減少使用者的疑惑。

✅ 前端 UI 正確範例：
- `14 日 RSI`
- `20 日乖離率`
- `9 日 KD 指標`

這項規範可確保無論是開發者還是 AI 助手，在新增任何金融專有名詞或技術指標時，都能自動產出整齊劃一的說明卡片。
