# 2026-03-05 日誌：K 線圖整合優化需求錄

根據今日討論，我們將於明日（2026-03-06）針對 `StockDetail` 頁面與 `TVChart` 元件進行 K 線圖顯示邏輯的深度優化。

## 優化目標
將「顯示時間範圍（Range）」與「K 線頻率（Interval）」解耦，並在 UI 上提供更專業且明確的切換功能與指標標注。

---

## 需求明細

### 1. UI 介面重構
-   **範圍 (Range) 切換：** 原有的「日、週、月、3月、1年」調整為橫向範圍選擇標籤（1D, 5D, 1M, 3M, 1Y）。
-   **頻率 (Interval) 切換：** 在圖標下方新增另一排小按鈕，供使用者自由選擇 K 線精度：
    -   `1m`, `5m`, `15m` (用於短線)
    -   `1h` (用於波段)
    -   `1d` (用於長期)
    -   `1wk`, `1mo` (用於大趨勢)

### 2. 後端與前端資料邏輯對接
-   **自動搭配 (預設)：** 如果使用者沒手動切換頻率，則依循建議的搭配（例如 1D 配 1m，5D 配 15m）。
-   **自由組合：** 若使用者手動點選，則需根據新的 Range 與 Interval 組合發送 API 請求。
    -   例：Range=1M, Interval=5m -> 請求 `/stocks/{id}/kline?period=1mo&interval=5m`

### 3. 技術指標內容標注化
-   **動態標題：** 技術指標卡片（MA20, RSI 等）需根據當前使用的 Interval 動態顯示名稱。
    -   例如在 1 分線下顯示 `MA20 (1分)`。
    -   例如在 日線下顯示 `MA20 (日線)`。
-   **說明補強：** 在指標說明中向使用者解釋不同頻率下指標代表的意義。

### 4. 其它細節
-   同步更新數據加載時的 Loading 狀態平滑度。
-   確保畫出的均線（MA）等技術指標能正確對應到當前 K 線頻率。
