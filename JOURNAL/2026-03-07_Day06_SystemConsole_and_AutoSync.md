# 🚀 AlphaForge 學習日誌 - Day 06

**主題**：自動化維護與系統透視 (System Observability)

---

## 🧠 今日學習重點 (The "Why")

### 1. 數據密集的 UI 哲學 (Data-Dense Design)
- **核心觀念**：對於專業量化工具，資訊的「完整性」與「掃描效率」優先於「留白美學」。
- **實踐技巧**：
    - **單行化 (One Row)**：強迫資訊對齊，減少眼球上下移動的負擔。
    - **色調收斂**：非關鍵資訊（時間、標籤）統一使用灰色，讓「成功（綠）」或「錯誤（紅）」的燈號能瞬間脫穎而出。
    - **響應式隱藏**：在手機端隱藏重複的日期，將珍貴的寬度留給核心訊息。

### 2. 系統可觀察性 (System Observability)
- **重要性**：後台排程（如 15:30 的爬蟲）如果沒有回饋，使用者會對系統產生不信任感。
- **解決方案**：實作 `SystemConsole`。這不只是日誌，而是系統的「心跳監測」。
- **互動設計**：加入「點擊外部自動縮小」，讓監控面板在不需要時能優雅地退回背景。

### 3. API 設計的魯棒性 (Robustness)
- **防禦性編程**：在前端實作 `Loading` 狀態防止「狂按 sync」導致的資源浪費。
- **冪等性 (Idempotency)**：確保爬蟲任務在重複執行時，能正確檢查資料庫是否已有資料並自動跳過。

---

## 🏗️ 開發與維運成果

### ✅ 自動化數據管線 (Data Pipeline)
- **官方 API 批發**：實作 `MarketDataCrawler` 直接下載 TWSE/TPEx 每日行情，效率提升 100 倍。
- **定時任務 (Scheduling)**：使用 APScheduler 設定每日盤後 15:30 自動啟動同步。

### ✅ 系統監控 UI (System Console)
- **終端機風格**：成功實作風格酷炫且資料度極高的日誌面板。
- **跨平台優化**：手動處理 CSS Grid 對齊與日期分隔線，達成手機與電腦完美兼容。
