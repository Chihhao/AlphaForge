# 🚀 AlphaForge 學習與開發計畫書

本計畫旨在透過開發 **AlphaForge** 平台，同步達成「掌握股票投資」、「精通量化分析」以及「實作 AI Agent」三大目標。

---

## 📅 階段一：建立盤感與指標拆解 (從 ECF 學習)
**目標：** 利用 ECF 工具作為「字典」，學習基本股票術語與 K 線型態。

*   **股市學習**：
    *   每天挑選 1-3 檔股票放入 ECF，觀察其「自動標註」的 K 線型態（如：吞噬、晨星）。
    *   透過 AI 問答，理解指標（MA, KD, RSI, MACD）的數學原理與物理意義。
*   **AlphaForge 開發**：
    *   實作「指標字典頁面」：將 ECF 的靜態指標轉化為 AlphaForge 的 UI 元件。
    *   串接基礎資料源：確保 Python 後端能穩定抓取 Yahoo Finance/TWSE 的歷史數據。
    *   **Day 3 實作**：在首頁加入「每日名詞解釋」小卡，讓使用者在看盤的同時學習指標背後的意義。
    *   **Day 6 實作**：建構 `SystemConsole` 監聽全市場盤後自動同步任務，實現「極致資料密度」的監控介面。
*   **AI Agent 學習**：
    *   學習如何給予 AI Agent (即 Antigravity) 明確的「指令規範」，讓 AI 協助生成指標計算代碼。

## 🗄️ 核心支線：量化資料庫進化史 (Data Pipeline Evolution)
**目標：** 建立穩定、高效且不依賴第三方即時 API 的量化選股資料基底，由淺入深實現真正的「大海撈針」(全台股 2000 檔瞬間掃描)。

*   **AlphaForge 開發**：
    *   **階段 A：預選池 MVP (Proof of Concept)**：
        *   **作法**：配置 30~50 檔權值/熱門股清單（如 0050 成分股），透過迴圈呼叫現有 API 抓取資料。
        *   **目的**：最快打通「指標計算 👉 API 傳輸 👉 前端雷達渲染」的全端架構。供初期驗證策略寫法，建立開發手感與成就感。
    *   **階段 B：每日盤後全市場批發 (Market Breadth) [DONE]**：
        *   **作法**：實作爬蟲，每日盤後一次性下載證交所/櫃買中心官方「每日收盤行情(MI_INDEX)」CSV 總表。
        *   **目的**：避開 API Rate Limit，達成一秒抓取全市場「當日」價格。讓雷達能顯示全市場當日的強弱勢與爆量排行。
    *   **階段 C：本地向量化歷史引擎 (Ultimate Quant Engine)**：
        *   **作法**：建立本地歷史 K 線庫（處理除權息等髒資料）。結合階段 B 每日更新，全面使用 Pandas 進行矩陣(Vectorization)運算。
        *   **目的**：終極型態。實現全台灣 2000 檔股票複雜技術指標（如均線糾結、長天期乖離）的毫秒級瞬間掃描。

## 📅 階段二：公式提煉與自訂化 (打造專屬武器庫)
**目標：** 在 AlphaForge 中實現讓使用者「自定義公式」的功能。

*   **股市學習**：
    *   學習「條件組合」：例如「KD 低檔 + 成交量翻倍」代表什麼意義？
    *   理解「濾網」觀念：如何剔除雜訊。
*   **AlphaForge 開發**：
    *   **Formula Builder**: 開發前端介面，讓使用者能選取指標並設定參數（如：`MA(5) > MA(20)`）。
    *   **Backend Calculator**: 在 Python 端動態解析這些公式，並計算出買賣訊號點。
*   **AI Agent 學習**：
    *   **代碼生成 (Code Generation)**：讓 AI Agent 根據使用者的白話文需求（如：「我想要找最近三天都漲連的股票」）自動轉換為 Python 邏輯。

## 📅 階段三：歷史回測與科學驗證 (真理檢驗)
**目標：** 建立回測引擎，用「數據」告訴你公式是否有用。

*   **股市學習**：
    *   學習「期望值」與「勝率」：理解為什麼高勝率不代表高獲利。
    *   認識「最大回撤 (MDD)」：學習控制風險。
*   **AlphaForge 開發**：
    *   開發 **Backtest Engine**：讓使用者選定一段歷史區間（如過去 5 年），模擬執行該公式。
    *   產出績效報表：包含總回饋率、勝率、賺賠比、資金曲線圖。
*   **AI Agent 學習**：
    *   **數據分析 Agent**：訓練 AI 如何分析回測結果，並主動給出優化建議（例如：「調整停損比例可以讓報酬率提高 10%」）。

## 📅 階段四：自動化通知與 AI Agent 進階化 (實戰自動導航)
**目標：** 系統 24 小時監控盤勢，主動通知買賣訊號。

*   **股市學習**：
    *   實戰心理學：學習在訊號出現時，如何克服恐懼或貪婪去執行。
*   **AlphaForge 開發**：
    *   **Scheduler**: 實作每日盤後自動掃描全台股任務。
    *   **Notification**: 整合 LINENotify 或 Telegram，當公式觸發訊號時主動推播。
*   **AI Agent 學習**：
    *   **自主 Agent (Autonomous Agent)**：實作一個會「自我檢討」的 Agent。它會觀察通知後的股價動向，並在隔日總結：「昨天的買入訊號正確/錯誤，原因可能是...」。

## 📅 階段五：另類數據與新聞情緒量化 (AI 情緒計分雷達 - 未來擴充)
**目標：** 突破傳統技術分析，利用大語言模型 (LLM) 對財經新聞進行自然語言處理，將主觀新聞轉化為客觀的「量化情緒權重」。

*   **股市學習**：
    *   學習「已反應 (Priced In)」現象與「買在謠言，賣在事實」的市場心理。
    *   理解「另類數據 (Alternative Data)」在現代量化交易中的價值與陷阱。
*   **AlphaForge 開發**：
    *   **Data Pipeline**: 開發每日新聞爬蟲，抓取特定個股相關的財經新聞。
    *   **Multi-Factor Model**: 在現有技術指標選股雷達上，新增「AI 新聞情緒分數」欄位，打造多因子(技術面+籌碼面+情緒面)選股模型。
*   **AI Agent 學習**：
    *   **NLP 評分 Agent**：設計精準的 Prompt，讓 AI 針對新聞內文萃取實體（股票名稱），並排除主觀臆測，依據營運/財報事實給予 -10 到 +10 的情緒評分。
