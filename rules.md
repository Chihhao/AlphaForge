# AlphaForge 開發規範 (Rules)

此文件用於記錄專案中獨特的開發約定、設計模式以及自定義的語法標準。

## 1. 知識庫與提示元件 (Glossary & EducationalHint)

為了保持 `glossary.json` 的整潔，避免混入大量冗長的 HTML `<span>`，我們在系統內定義了專屬的 Markdown 微語法。這些語法會在前端的 `EducationalHint` 元件中透過 `rehype-raw` 與 `replace` 進行預防性轉換，提供具有強烈**金融情緒色彩**的視覺回饋。

### 語法定義

| 語法 | 意義 / 情境 | 渲染樣式 | 範例 |
| :--- | :--- | :--- | :--- |
| `**關鍵字**` | 中性的名詞強調、公式變量、重要狀態。 | **亮金色粗體** (`text-gold-400 font-bold`) | `**大成交量**`、`**K > 80**` |
| `++正向字++` | 多頭、買進訊號、利多消息、強勢狀態。 | **翠綠色粗體** (`text-emerald-400 font-bold`) | `++黃金交叉++`、`++強勢訊號++` |
| `--負向字--` | 空頭、賣出訊號、利空消息、弱勢與風險。 | **玫瑰紅粗體** (`text-rose-400 font-bold`) | `--死亡交叉--`、`--追高風險--` |

### 撰寫原則
1. **保持精簡**：說明文字請盡量簡短，重點在於公式以及「什麼情況代表什麼事」。
2. **善用 Markdown 標記**：標題統一使用 `## 標題`，公式使用 LaTeX 語法 `$$ ... $$` 或是 `$ ... $`。
3. **拒絕 Inline HTML**：嚴禁在字典 JSON 檔中寫死 ` <span class="..."> `，以保持原始文字檔的純粹與可維護性。

## 2. UI 設計原則
1. **一體化深色主題**：遵循由 Tailwind 構成的深色玻璃擬態 (Glassmorphism) 主題，背景通常以 `bg-gray-800` 或是 `zinc-900` 為主。
2. **顏色語意嚴謹制**：金色代表提示與重點，綠色僅限用於正向報酬/看多訊號，紅色僅限用於負向報酬/看空訊號。
