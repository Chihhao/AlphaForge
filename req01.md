# 台灣股市分析與模擬交易平台 - 需求規格書 (Draft v0.1)

## 1. 專案概述 (Project Overview)
建立一個網頁應用程式，提供使用者查看台灣股市（TW Stock）的即時/歷史行情，並具備技術分析圖表功能。同時，提供模擬交易系統，讓使用者在不承擔真實資金風險的情況下驗證交易策略。

## 2. 核心功能需求 (Functional Requirements)

### 2.1 市場數據模組 (Market Data)
*   **股票搜尋**：支援輸入代號或名稱搜尋台股上市櫃股票。
*   **K線圖表**：提供日K、週K、月K，以及分時走勢圖（Intraday）。
*   **基本資訊**：顯示開盤、收盤、最高、最低、成交量、漲跌幅。
*   **資料來源**：需介接真實台股數據（待確認來源：證交所爬蟲、Yahoo Finance API、或券商 API 如 Fugle/Shioaji）。

### 2.2 技術分析模組 (Technical Analysis)
*   **主圖指標**：移動平均線 (MA)、布林通道 (Bollinger Bands)。
*   **副圖指標**：RSI、MACD、KD (Stochastic Oscillator)、成交量 (Volume)。
*   **互動功能**：使用者可在圖表上進行縮放、平移。

### 2.3 模擬交易模組 (Paper Trading)
*   **虛擬帳戶**：每個使用者註冊後獲得一筆虛擬初始資金（例如：100萬 TWD）。
*   **下單功能**：
    *   支援「市價單」與「限價單」。
    *   支援「買入」與「賣出」。
*   **庫存管理**：即時顯示持有股數、平均成本、現價、未實現損益。
*   **交易紀錄**：完整的歷史委託與成交紀錄。
*   **績效分析**：計算總資產變化、報酬率。

### 2.4 會員系統 (User System)
*   註冊/登入 (Email 或 Google OAuth)。
*   個人設定頁面。

## 3. 非功能性需求 (Non-functional Requirements)
*   **響應式設計 (RWD)**：需同時支援桌機與手機瀏覽。
*   **效能**：K線圖表載入需流暢，數據延遲需在可接受範圍內。
*   **擴充性**：後端架構需預留未來加入「程式交易回測」功能的空間。

## 4. 待討論與確認事項 (To Be Discussed)

為了讓規格更具體，我們需要確認以下技術決策：

### 4.1 數據源 (Data Source)
*   **選項 A (開源套件組合)**：
    *   歷史資料：使用 `yfinance` (Yahoo Finance)，免費且容易取得長區間 K 線資料。
    *   即時資料：使用 `twstock` (爬蟲證交所)，可取得即時報價，但需注意頻率限制。
    *   *優點：完全免費，無需註冊。缺點：穩定性較差。*
*   **選項 B (專業 API)**：
    *   使用 `Fugle` (富果) 或 `FinMind` API。
    *   *優點：資料穩定、支援 WebSocket 即時推播、有五檔報價（對模擬撮合很重要）。缺點：需註冊/申請 Token。*
*   **選項 C (中國信託 API)**：
    *   支援 Python，但申請門檻較高（通常需有一定交易量實績才能申請），且文件較不公開。
    *   *現況：因開發環境為 Mac M1，不支援 Windows COM 元件，故放棄。*
*   **決策**：確認使用 **選項 A (開源套件組合)** 進行開發。

### 4.2 技術堆疊 (Tech Stack)
*   **前端**：建議使用 React 或 Next.js，搭配圖表庫 (如 TradingView Lightweight Charts 或 Highcharts)。
*   **後端**：
    *   *Python (FastAPI/Django)*：金融分析首選，豐富的 Pandas/Numpy 生態系，計算指標方便。
    *   *Node.js*：適合高併發 I/O，但數據處理能力較弱。
    *   **建議**：Python (FastAPI) + PostgreSQL (資料庫)。

### 4.3 模擬撮合機制
*   模擬交易要做到多細緻？
    *   **簡單版**：以當下「最新成交價」直接成交。
    *   **擬真版**：需考慮「五檔報價」的量，若模擬買入量大於掛單量，需分批成交 (滑價模擬)。
